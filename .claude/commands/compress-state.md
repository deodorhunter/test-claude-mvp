<identity>
Context compression command. Synthesizes everything important from the current session into a compact portable snapshot. Operates entirely from in-context information — no file reads. After writing the snapshot, instructs the user to /clear and resume in a fresh context.
</identity>

<hard_constraints>
1. IN-CONTEXT ONLY: Do NOT read any files to build the snapshot. Synthesize from conversation history only.
2. INTENTIONAL OVERWRITE: docs/.temp_context.md is the ONE file where `>` overwrite is correct. Always the latest snapshot.
3. NO CODE IN SNAPSHOT: Only decisions, status, and the resume prompt. Never paste source code.
4. SIZE LIMIT: Snapshot must be under 100 lines — a pointer, not a transcript.
5. NO SESSION SYNC: This file stays in the project directory. Never commit it or sync it externally (rule-011).
</hard_constraints>

<workflow>
1. From conversation history, extract:
   - Current phase and sub-phase
   - Last completed US, in-progress US, next US
   - Open blockers
   - Key decisions made this session (with 1-line rationale)
   - BACKLOG status summary (from memory, not file read)
   - Files created/modified this session
2. Write `docs/.temp_context.md` (intentional overwrite):

```markdown
# Context Snapshot
> Generated by /compress-state on [DATE]
> To resume: read this file, then follow "Next Action" below.

## Session Summary
[2-3 sentences: what was accomplished]

## Current Phase & Status
- **Phase:** [e.g. Phase 2d]
- **Last completed US:** [US-NNN — Title | ✅ Done]
- **In progress:** [US-NNN — Title | status]
- **Next:** [US-NNN — Title]
- **Awaiting user approval:** [yes/no — for what?]

## Open Blockers
[List or "None"]

## Key Decisions
1. [Decision] — [rationale]

## BACKLOG Status
| US | Title | Status |
|---|---|---|
[from memory]

## Files Created/Modified This Session
- [path] — [what changed]

## Next Action
**Resume prompt:** "[exact prompt to type to resume]"

1. Type /clear to reset context window
2. Start new chat
3. Type: "Read docs/.temp_context.md and resume from where we left off."
```

3. Display to user: snapshot written + 2-line summary + ACTION REQUIRED instructions.
</workflow>

<output_format>
Output EXACTLY:
✅ Context snapshot written to docs/.temp_context.md

📋 Captured: [2-line summary of snapshot content]

⚠️ ACTION REQUIRED:
1. Type /clear now to reset context window
2. Start a new chat
3. Type: "Read docs/.temp_context.md and resume"

Token savings from /clear on a long session: typically 15,000–40,000 input tokens.
</output_format>
