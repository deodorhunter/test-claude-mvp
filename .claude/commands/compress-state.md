# Command: /compress-state
> Summarize the current chat session into a portable context snapshot.
> Use when the conversation is getting long (> 20 messages) or token usage is climbing.
> After running this command, type `/clear` to reset the context window.
> Invocation: type `/compress-state` in the chat.

---

## What This Command Does

1. Synthesizes everything important from the current conversation into a compact snapshot
2. Writes the snapshot to `docs/.temp_context.md` using `>` (intentional full overwrite — this is always the latest snapshot)
3. Instructs the user to type `/clear` to wipe the context window
4. The next session starts fresh by reading `docs/.temp_context.md` to resume exactly where you left off

---

## Instructions for Claude

Execute these steps immediately when `/compress-state` is invoked.

### Step 1 — Synthesize the Session

Without reading any files (this is an in-context operation only), extract from the conversation:

**Current Phase & Status:**
- Which phase/sub-phase are we in?
- Which US was last completed? Which is in progress? Which is next?
- What was the last decision made?

**Open Items:**
- Any blockers reported?
- Any US waiting for user approval?
- Any unresolved questions?

**Key Decisions Made This Session:**
- Architectural decisions (list with 1-line rationale each)
- Model selections made (which US used which model)
- Any deviations from the standard workflow (and why)

**State of Key Files:**
- What is the current content of `docs/backlog/BACKLOG.md` (summarize the status table, don't read the file — recall from conversation)
- What US files were created or updated this session?

**Next Action:**
- Exactly what should the next session start with?
- What command or prompt should the user type to resume?

### Step 2 — Write the Snapshot

Write `docs/.temp_context.md` using the Write tool (intentional `>` overwrite — this is the snapshot, not append-only):

```markdown
# Context Snapshot
> Generated by /compress-state on [DATE] at [approximate time in session]
> To resume: start a new chat, read this file, then follow "Next Action" below.

## Session Summary
[2-3 sentences: what was accomplished this session]

## Current Phase & Status
- **Phase:** [e.g. Phase 2a — Plugin System]
- **Last completed US:** [US-NNN — Title | ✅ Done / ⚠️ Blocked]
- **In progress US:** [US-NNN — Title | status]
- **Next US:** [US-NNN — Title]
- **Awaiting user approval:** [yes/no — for what?]

## Open Blockers
[List any blockers from the session, or "None"]

## Key Decisions Made
1. [Decision] — [rationale]
2. ...

## BACKLOG Status (summary)
| US | Title | Status |
|---|---|---|
[fill from conversation memory]

## Files Created/Modified This Session
- [file path] — [what changed]
- ...

## Next Action
> **Resume prompt:** "[exact prompt the user should type to resume work]"

**Step 1:** Type `/clear` in Claude Code to reset the context window.
**Step 2:** Start a new chat.
**Step 3:** Type: "Read docs/.temp_context.md and resume from where we left off."
```

### Step 3 — Instruct the User

After writing the file, display this message:

```
✅ Context snapshot written to docs/.temp_context.md

📋 Summary of what was captured:
- [2-line summary of the snapshot content]

⚠️  ACTION REQUIRED:
1. Type /clear now to reset the context window and save tokens
2. Start a new chat
3. Open docs/.temp_context.md and tell Claude: "Read this and resume"

This snapshot will let the next session resume exactly where you left off.
Token savings from /clear on a long session: typically 15,000–40,000 input tokens.
```

---

## Hard Constraints

- This command operates entirely from in-context information — do NOT read files to build the snapshot
- `docs/.temp_context.md` is the ONE file where intentional full overwrite (`>`) is correct — it is always the latest snapshot
- Do NOT include code in the snapshot — only decisions, status, and the resume prompt
- Keep the snapshot under 100 lines — it's a pointer, not a transcript
- Do NOT append to `docs/ARCHITECTURE_STATE.md` as part of this command — that happens via `/handoff`
