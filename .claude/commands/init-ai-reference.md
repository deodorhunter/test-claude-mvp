# Command: /init-ai-reference
> Scan the repository ONCE and write the static `docs/AI_REFERENCE.md` file.
> Run this once per project setup, and again after major stack changes.
> Invocation: type `/init-ai-reference` in the chat.

---

## What This Command Does

You are about to perform the ONE allowed exploration of this repository.
This command grants a **single, time-boxed scan** to produce a static reference file.
After this command completes, all future Claude sessions read `docs/AI_REFERENCE.md` instead of exploring.

---

## Instructions for Claude

Execute these steps in order. Use the results to write `docs/AI_REFERENCE.md`.

### Step 1 — Single Repo Scan (use Bash, ONE command)

```bash
find . -maxdepth 3 \
  \( -name "*.toml" -o -name "*.json" -o -name "Makefile" \
     -o -name "docker-compose*.yml" -o -name "requirements*.txt" \
     -o -name "package.json" -o -name ".env.example" \
     -o -name "alembic.ini" -o -name "pytest.ini" \
     -o -name "tsconfig.json" -o -name "vite.config.*" \) \
  -not -path "*/node_modules/*" \
  -not -path "*/.git/*" \
  -not -path "*/dist/*" \
  -not -path "*/__pycache__/*" \
  2>/dev/null | sort
```

### Step 2 — Read Key Config Files (targeted, max 5 files)

From the scan output, read only:
- `Makefile` (to extract make targets)
- `docker-compose.yml` or `docker-compose.local.yml` (to extract services and ports)
- `pyproject.toml` or `requirements.txt` (to identify Python stack)
- `package.json` at root or `frontend/` (to identify JS stack)
- `.env.example` (to list env var names)

Do NOT read application code files. Do NOT read test files. Do NOT read migration files.
