# US-014: MCP Registry + Trust Scoring

**Agente:** AI/ML Engineer
**Fase:** 2c
**Dipendenze:** US-005
**Priorità:** high
**Stato:** 📋 Backlog

## Contesto

Il sistema integra MCP (Model Context Protocol) servers come fonti di contesto aggiuntivo. Ogni server ha un trust score (0–1) che determina se i suoi risultati vengono inclusi nel contesto. Risultati sotto soglia minima vengono filtrati.

Trust scores MVP:
- `internal_docs`: 0.95
- `official_api`: 0.9
- `github`: 0.7
- `web`: 0.4

## Task

Implementare `MCPRegistry` in `ai/mcp/registry.py`:
- Registro di MCP servers disponibili con i loro trust scores
- `MCPServer` base class con metodo `async query(input_text) → {data, source, confidence}`
- Implementazione stub per `internal_docs` e `web` (risposte fisse per demo)
- Filtraggio risultati sotto `min_confidence` (configurabile, default 0.5)
- Audit log entry per ogni MCP query

## Acceptance Criteria

- [ ] `MCPRegistry` registra server e li recupera per nome
- [ ] `MCPServer.query()` ritorna `{data, source, confidence}`
- [ ] Risultati con `confidence < min_confidence` vengono filtrati (non inclusi nel contesto)
- [ ] Almeno 2 stub server implementati (`internal_docs`, `web`)
- [ ] Audit log entry per ogni query MCP (action: `mcp_query`)
- [ ] Unit test: registry lookup, trust filtering, audit log trigger
- [ ] Completion summary in `docs/progress/US-014-done.md`
