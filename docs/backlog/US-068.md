## US-068: Pre-Collect Command Metadata for Agent Delegation

**Agent:** Product Owner
**Phase:** 3b (Automation & Cognitive Tooling)
**Dependencies:** US-054 (generates command catalog)
**Priority:** medium
**Status:** 📋 Backlog

## Context

Phase 3a: US-053 delegation required injecting a catalog of 12 slash commands. The command list was generated via bash at task time (~3k tokens). US-054 will audit `.claude/commands/` and generate a catalog; US-068 takes that output and pre-collects it in a reference file for reuse across all future agent delegations.

## Task

1. Create a reference file: `docs/.command-catalog.md` or similar
2. Generate once: run `find .claude/commands -name "*.md" -exec echo "{}" \;` to discover all commands
3. For each command: extract name and 1-line description (from `description:` field in frontmatter)
4. Format as a simple table or list (keep <100 lines)
5. Update delegation prompts (e.g., US-053 template in `.claude/agents/doc-writer.md`) to reference this file instead of re-generating at delegation time
6. Document in "Orchestration Patterns" section of `docs/AI_REFERENCE.md`

## Acceptance Criteria

- [ ] Command catalog file created and maintained
- [ ] All 12+ project commands listed with 1-line descriptions
- [ ] File is under 100 lines (reference density)
- [ ] Delegation prompt templates updated to reference (not re-read) this file
- [ ] Estimated savings: ~3k tokens per agent delegation using command reference

## Files Involved

```
docs/.command-catalog.md (new reference)
.claude/agents/doc-writer.md (update delegation template)
docs/AI_REFERENCE.md (reference in Orchestration Patterns)
```

## Expected Output

Reusable command catalog reference for all future agent delegations
