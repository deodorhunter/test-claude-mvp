# US-012: Model Layer — Ollama + Claude Adapters, generate() Interface

**Agente:** AI/ML Engineer
**Fase:** 2b
**Dipendenze:** US-002
**Priorità:** critical
**Stato:** 📋 Backlog

## Contesto

Il sistema supporta due provider AI nell'MVP:
- **Ollama** (demo mode): modello locale nel container Docker, nessuna API key richiesta. Usato di default.
- **Claude via Anthropic API** (demo-api mode): richiede `ANTHROPIC_API_KEY` nel `.env`. Attivato tramite config.

Altri provider (GitHub Copilot, ecc.) sono **mockati** e non implementati nell'MVP.

La modalità è determinata da `AI_MODE` in `.env` (`demo` | `demo-api`).

## Task

Implementare il model layer in `ai/models/`:
- Interfaccia base `ModelAdapter` con metodo `async generate(prompt: str, context: str) → ModelResponse`
- `OllamaAdapter`: chiama Ollama HTTP API (`http://ollama:11434`), modello configurabile (default: `llama3`)
- `ClaudeAdapter`: chiama Anthropic API con `ANTHROPIC_API_KEY`, modello `claude-haiku-4-5-20251001` (più economico per demo)
- `MockAdapter`: risposta fissa per test — sempre disponibile
- Factory `get_model_adapter(config) → ModelAdapter` che seleziona il provider corretto
- `ModelResponse`: `{text, model_used, tokens_used, provider}`

## Acceptance Criteria

- [ ] `ModelAdapter` base implementata con interfaccia `generate(prompt, context)`
- [ ] `OllamaAdapter` chiama `http://ollama:11434/api/generate` e ritorna `ModelResponse`
