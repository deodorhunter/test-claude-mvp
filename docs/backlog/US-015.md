# US-015: Context Builder — MCP Query, Filter, Source Attribution

**Agente:** AI/ML Engineer
**Fase:** 2c
**Dipendenze:** US-014
**Priorità:** high
**Stato:** 📋 Backlog

## Contesto

Il Context Builder assembla il contesto finale per il modello combinando output di MCP servers, plugin, e RAG. Ogni chunk di contesto deve includere source attribution e confidence. L'output deve essere sanitizzato per prevenire prompt injection.

## Task

Implementare `ContextBuilder` in `ai/context/builder.py`:
- Query parallela a tutti i MCP server registrati (asyncio.gather)
- Filtraggio per trust score (delega a MCPRegistry)
- Formattazione: `[Fonte: {source} | confidence: {confidence}]\n{data}`
- Sanitizzazione output: strip HTML, blocca pattern di prompt injection (`IGNORE PREVIOUS`, `[INST]`, ecc.)
- Combinazione con RAG results (US-016) e plugin output

## Acceptance Criteria

- [ ] `ContextBuilder.build(query, tenant_id)` ritorna stringa di contesto formattata
- [ ] Ogni chunk include `[Fonte: X | confidence: Y]` prefix
- [ ] Output sanitizzato: prompt injection patterns rilevati e rimossi (log warning)
- [ ] Query MCP servers in parallelo (asyncio.gather) con timeout 3s per server
- [ ] Server MCP che non risponde entro 3s → skippato (non errore fatale)
- [ ] Unit test: context assembly, sanitization, timeout handling, source attribution
- [ ] Completion summary in `docs/progress/US-015-done.md`
