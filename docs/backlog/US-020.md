# US-020: plone-mcp — Self-Hosted MCP Server + Python Adapter

**Agente:** DevOps/Infra (Docker) + AI/ML Engineer (Python adapter)
**Fase:** 2d (addition)
**Dipendenze:** US-014 (MCP Registry)
**Priorità:** high
**Stato:** ✅ Done

## Contesto

Scoperta di https://github.com/plone/plone-mcp — server MCP Node.js ufficiale Plone che espone
content management tools (search, create, update, workflow). Il progetto usa stdio transport,
nessun Docker, nessun npm registry. Dobbiamo self-hostarlo prima di chiudere Phase 2.

Architettura: plone-mcp wrappato con SSE transport → Docker service porta 9120 →
Python `PloneMCPServer(MCPServer)` adapter lo interroga via HTTP.

## Task

1. **Docker service** (`plone-mcp`):
   - Clona plone-mcp in `infra/plone-mcp/`
   - Aggiunge `infra/plone-mcp/sse-server.ts` (~30 righe): Express + SSEServerTransport
   - Crea `infra/docker/Dockerfile.plone-mcp` (Node.js 20-alpine, multi-stage build)
   - Aggiunge servizio in `infra/docker-compose.yml` (port 9120, dipende da `plone`)

2. **Python adapter** (`ai/mcp/servers/plone.py`):
   - `PloneMCPServer(MCPServer)`: chiama `http://plone-mcp:9120` via HTTP/SSE
   - Invia `plone_search` tool call → parsa risposta → `MCPResult`
   - Output sempre passato per `ai/context/sanitizer.sanitize()`
   - `trust_score = 0.85` (first-party CMS)

3. **MCP Allowlist** in `ai/mcp/registry.py`:
   - Aggiunge `MCP_ALLOWLIST = {"plone", "internal_docs", "web"}`
   - Validazione all-field schema (rule-012, FSP protection)

4. **Env vars** in `.env.example`:
   - `PLONE_USERNAME`, `PLONE_PASSWORD`, `PLONE_MCP_URL`

5. **Test** (`backend/tests/test_plone_mcp.py`):
   - Happy path, HTTP 4xx error handling, timeout, sanitization, trust_score

## Acceptance Criteria

- [ ] `docker compose up` include servizio `plone-mcp` funzionante su porta 9120
- [ ] `GET http://localhost:9120/sse` ritorna 200 (healthcheck)
- [ ] `PloneMCPServer.query("test")` ritorna `MCPResult(data=..., source="plone:...", confidence=0.85)`
- [ ] Output MCP passato per sanitizer (test verifica injection strippata)
- [ ] `MCP_ALLOWLIST` presente in `registry.py`; server non in allowlist → non registrabile
- [ ] Tutti i test in `test_plone_mcp.py` verdi
- [ ] `PLONE_USERNAME`, `PLONE_PASSWORD`, `PLONE_MCP_URL` in `.env.example`
- [ ] Security Engineer sign-off: credentials non loggati, nessun cross-tenant leakage
