# US-011: Plugin Runtime — Subprocess Isolation + Resource Limits

**Agente:** Security Engineer
**Fase:** 2a
**Dipendenze:** US-010
**Priorità:** critical
**Stato:** ✅ Done

## Contesto

I plugin vengono eseguiti in subprocess isolati per prevenire che un plugin malfunzionante o compromesso possa accedere ai dati di altri tenant o all'host. Comunicazione via JSON su stdin/stdout. Un pool di subprocess per coppia `(tenant_id, plugin_name)`.

**Eccezione MVP:** `plone_integration` ha accesso di rete a Plone — tutti gli altri plugin hanno rete bloccata.

## Task

Implementare `PluginRuntime` in `backend/app/plugins/runtime.py`:
- Esecuzione plugin in subprocess con timeout 10s
- Limiti CPU/memoria tramite resource limits (cgroups se disponibili, fallback `resource` module)
- Comunicazione JSON su stdin/stdout
- Pool di subprocess per `(tenant_id, plugin_name)` con max 2 worker per coppia
- Network access bloccato per tutti i plugin eccetto `plone_integration`

## Acceptance Criteria

- [x] Plugin eseguito in subprocess separato (non in-process)
- [x] Timeout 10s: plugin che supera il limite viene killato e ritorna PluginTimeoutError
- [x] Limite memoria applicato (RLIMIT_AS 256MB; warning esplicito se non supportato)
- [x] Input/output solo via JSON su stdin/stdout — nessun altro canale
- [x] Plugin di tenant A non può leggere dati di tenant B (verifica via test)
- [x] Env vars isolate (subprocess riceve solo PATH e PYTHONPATH — no DB_URL, SECRET_KEY, ecc.)
- [x] Unit test: timeout enforcement, isolation cross-tenant, JSON protocol, missing entrypoint, bad JSON
- [x] Security Notes in `docs/progress/US-011-done.md` con rischi residui HIGH/MEDIUM/LOW espliciti
- [x] Network isolation: non mitigata a livello kernel (MVP — documentato come RISK-HIGH)
