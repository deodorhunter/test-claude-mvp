# US-070: Accuracy Scoring — Structured /judge Output + Trend Log

**Agent:** Doc Writer
**Phase:** 3d (Infrastructure & Architecture)
**Dependencies:** US-069 (gitignore entries already added)
**Priority:** medium
**Status:** 📋 Backlog

## Context

The `/judge` skill currently prints a human-readable pass/fail verdict per US acceptance criterion. This output is ephemeral — it exists in the conversation but is never stored, so we have no trend data on: how many ACs pass first-time, QA bounce-back rate per agent type, or accuracy improvement/regression across phases.

US-070 adds a structured append to `benchmark/accuracy-log.jsonl` at the end of each `/judge` run, enabling trend analysis over time.

## Task

1. **Update** `.claude/commands/judge.md`:
   - After printing the human-readable verdict, append a JSON line to `benchmark/accuracy-log.jsonl`:
     ```json
     {"date": "YYYY-MM-DD", "us": "US-NNN", "agent": "agent-type", "ac_total": N, "ac_pass": N, "ac_fail": N, "verdict": "pass|fail", "phase": "3d"}
     ```
   - Use `echo '...' >> benchmark/accuracy-log.jsonl` (append-only, never overwrite)
   - If `benchmark/accuracy-log.jsonl` doesn't exist, create it on first write

2. **Write** `benchmark/report-accuracy.sh`:
   - Read `benchmark/accuracy-log.jsonl`
   - Print per-phase accuracy summary: total ACs evaluated, pass rate %, bounce-back count (entries where verdict=fail)
   - Print per-agent-type breakdown
   - Flag any agent with pass rate < 80% as ⚠️

3. **Add** `make benchmark-accuracy` target.

4. **Update** `benchmark/README.md` with new section "Accuracy Tracking" explaining the log format and make target.

## Acceptance Criteria

- [ ] `.claude/commands/judge.md` appends a valid JSON line to `benchmark/accuracy-log.jsonl` after each run
- [ ] JSON line contains all required fields: date, us, agent, ac_total, ac_pass, ac_fail, verdict, phase
- [ ] `make benchmark-accuracy` prints per-phase and per-agent pass rate table
- [ ] `benchmark/accuracy-log.jsonl` is gitignored (handled by US-069)
- [ ] Append-only — no existing entries are modified or deleted
- [ ] `benchmark/README.md` documents the new target
