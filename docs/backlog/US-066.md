## US-066: Configure Serena MCP to Ignore .git/ Paths

**Agent:** DevOps/Infra
**Phase:** 3b (Automation & Cognitive Tooling)
**Dependencies:** rule-019
**Priority:** medium
**Status:** 📋 Backlog

## Context

Phase 3a retrospective extracted rule-019 (Serena-Git Isolation) due to 3 recurrences of git index.lock contention during worktree merges. Root cause: Serena LSP holds `.git/index` lock during parallel reads → "Unable to create index.lock" errors.

## Task

Configure Serena MCP in `settings.json` to ignore `.git/` and related paths:

1. Add or update `serena.ignored_paths` in `.claude/settings.json` (if file exists) or user's Claude Code settings
2. Include: `.git/`, `.git/**`, `node_modules/`, `__pycache__/`, `.venv/`
3. Document the configuration in `docs/AI_REFERENCE.md` under a new "IDE & LSP Settings" section (or append to Model Routing section)
4. Verify: restart Serena LSP and confirm index.lock no longer appears during git merge in worktrees

## Acceptance Criteria

- [ ] Serena `ignored_paths` configured in settings.json or user config
- [ ] `.git/` is in the ignore list
- [ ] Configuration documented in docs/AI_REFERENCE.md
- [ ] Manual test: git merge in worktree does not produce index.lock errors
- [ ] rule-019 reference documented
