## Context

User feedback: "not clear how to use or automate learn, judge, notepad, deep-interview." These patterns are referenced in the framework (`.claude/commands/`) but lack documentation connecting them to concrete workflows.

**Claude Code focus:** All 5 patterns already exist as Claude Code slash commands in `.claude/commands/`. This US documents their purpose, maps them to workflows, and evaluates automation potential.

## Task (Claude Code-first)

1. **Audit** existing `.claude/commands/`: learn.md, judge.md, notepad.md, reflexion.md, deep-interview.md
2. **Classify** each pattern:
   - **Implemented:** map to existing `.claude/commands/` file + describe trigger conditions
   - **Automatable:** can be hooked into `.claude/hooks/` (e.g., reflexion after phase gate)
   - **Manual-only:** requires human judgment to invoke
3. **Create** `docs/COGNITIVE_PATTERNS.md` with:
   - Definition of each pattern (1-2 sentences)
   - Claude Code command: `/learn`, `/judge`, etc.
   - When to use (trigger conditions)
   - Token cost estimate per invocation
   - Automation potential (hook-able vs manual)
4. **Update** `AI_PLAYBOOK.md` to reference the new doc
