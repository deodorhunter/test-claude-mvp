# US-010: Plugin Manager — Hot-plug, Manifest, Tenant Isolation

**Agente:** Backend Dev
**Fase:** 2a
**Dipendenze:** US-005
**Priorità:** critical
**Stato:** ✅ Done

## Contesto

Il sistema richiede un plugin manager che permetta di abilitare/disabilitare plugin per tenant a runtime, senza riavvio globale. I plugin sono definiti da un manifest YAML (`manifest.yaml`) e sono istanziati per-tenant. La lista di plugin abilitati è in `tenant_plugins` (DB).

I 4 plugin dell'MVP sono: `plone_integration`, `docs_rag`, `block_builder`, `doc_ingestion` — tutti pre-approvati (MVP: trusted, no sandbox user-upload).

## Task

Implementare `PluginManager` in `backend/app/plugins/manager.py`:
- Caricamento plugin da `manifest.yaml` (id, version, capabilities, entrypoint)
- Registro dei plugin attivi per tenant: `get_active_plugins(tenant_id)` → lista plugin abilitati
- Enable/disable plugin per tenant (aggiorna `tenant_plugins` in DB)
- Validazione manifest all'avvio
- Isolamento: un plugin non può accedere ai dati di un altro tenant

## Acceptance Criteria

- [x] `PluginManager` carica manifest da disco all'avvio
- [x] `get_active_plugins(tenant_id)` ritorna solo i plugin abilitati per quel tenant
- [x] Enable/disable plugin aggiorna DB `tenant_plugins` e si riflette immediatamente (no restart)
