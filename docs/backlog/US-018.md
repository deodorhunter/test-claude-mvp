# US-018: Security Review — Plugin Isolation + MCP Sanitization + Audit Integration

**Agente:** Security Engineer
**Fase:** 2d
**Dipendenze:** US-011, US-015
**Priorità:** critical
**Stato:** 📋 Backlog

## Contesto

Review di sicurezza su tutto il lavoro di Phase 2 prima del Phase Gate. Comprende:
1. Verifica isolamento plugin (US-010, US-011)
2. Verifica sanitizzazione MCP output (US-015)
3. Integrazione audit log: cablare gli eventi rimasti aperti da Phase 1 (auth events, RBAC denials) + nuovi eventi Phase 2

## Task

1. **Plugin isolation review**: verificare che subprocess isolation e resource limits siano corretti. Scrivere test che tentano di aggirare l'isolation.
2. **MCP sanitization review**: verificare che `ContextBuilder` blocchi prompt injection patterns. Aggiungere pattern mancanti se trovati.
3. **Audit integration**: cablare `AuditService` nei punti rimasti aperti:
   - Auth events in `backend/app/auth/` (login success/failure)
   - RBAC denials in `backend/app/rbac/middleware.py`
   - Quota exceeded in `backend/app/quota/`
4. **Refresh token rotation**: implementare rotation per prevenire token theft (residual risk US-004)

## Acceptance Criteria

- [ ] Test che dimostrano che plugin di tenant A non accede a dati di tenant B (eseguibili, passanti)
