# US-069: In-House Session Cost Accuracy + Drift Detection

**Agent:** Backend Dev
**Phase:** 3d (Infrastructure & Architecture)
**Dependencies:** —
**Priority:** high
**Status:** 📋 Backlog

## Context

Our SESSION_COSTS.md estimates are rough hand-written guesses. The ArkNill claude-code-cache-analysis repo found that local token accounting can be off by up to 2.87x due to log inflation from extended thinking, and that tool result truncation (Budget Cap bug) causes silent context drift after 15–20 tool uses — both undetectable without real data.

Claude Code already writes session JSONL to `~/.claude/projects/*/` with exact per-request token breakdowns (`input_tokens`, `output_tokens`, `cache_creation_input_tokens`, `cache_read_input_tokens`). No proxy required; all data stays local.

## Task

1. **Write** `benchmark/capture-session-metrics.sh`:
   - Locate the most recent Claude Code session JSONL under `~/.claude/projects/$(pwd | sed 's|/|-|g')/`
   - Extract per-request token fields using `jq`
   - Aggregate: total input, total output, total cache_read, total cache_creation, estimated cost (using Anthropic published rates for claude-sonnet-4-6 and claude-haiku-4-5)
   - Detect drift signals: count entries where tool result content length < 100 chars (Budget Cap proxy), count `compress-state` invocations
   - Output: `benchmark/session-metrics/YYYY-MM-DD-HH-MM.json` (structured, gitignored)

2. **Add** `make benchmark-session` target to `infra/Makefile` (or root `Makefile` if present):
   ```
   benchmark-session:
       @bash benchmark/capture-session-metrics.sh
   ```

3. **Write** `benchmark/report-session-costs.sh`:
   - Read all JSON files in `benchmark/session-metrics/`
   - Print a summary table: date, session, real_input_tokens, real_output_tokens, cache_read_ratio, estimated_cost_usd, drift_signals
   - Compare estimated_cost_usd against the last matching row in `docs/SESSION_COSTS.md` → print delta %

4. **Add** `make benchmark-report` target.

5. **Update** `.gitignore` to include `benchmark/session-metrics/` and `benchmark/accuracy-log.jsonl`.

6. **Update** `benchmark/README.md` with new section "Session Cost Capture" explaining the approach, data format, and make targets.

## Acceptance Criteria

- [ ] `benchmark/capture-session-metrics.sh` runs without error on a real session JSONL
- [ ] Output JSON contains: `session_date`, `input_tokens`, `output_tokens`, `cache_read_tokens`, `cache_creation_tokens`, `cache_read_ratio_pct`, `estimated_cost_usd`, `drift_signals_count`

- [ ] No data leaves the local machine (EU Act rule-011 compliance)
