# US-056: Automated Context Compression Hook (rule-010 Automation)

**Agent:** AI/ML Engineer
**Phase:** 3b (Automation & Cognitive Tooling)
**Dependencies:** US-051 (model routing informs compression thresholds)
**Priority:** medium
**Status:** 📋 Backlog

## Context

User feedback: "automate some things like context state compression and reflexion."
Rule-010 mandates `/compress-state` before parallel waves, but the trigger is entirely manual. Users forget it under task pressure, causing 240k+ token waste per parallel wave.

**Claude Code focus:** This is a Claude Code-only feature. Claude Code supports hooks registered in `.claude/settings.json` under `hooks` key. Verified event types: `UserPromptSubmit` (fires before prompt processing) and `PostToolUse` (fires after tool execution). Existing scripts: `.claude/hooks/pre-prompt.sh` and `.claude/hooks/post-tool-truncate.sh`. Copilot has no equivalent — compression is not applicable to Copilot's stateless chat model.

## Task

1. **Research** all available Claude Code hook event types beyond the two verified ones (`UserPromptSubmit`, `PostToolUse`). Do NOT assume other types exist — verify from docs or experimentation first.
2. **Define** automatic compression triggers:
   - After N tool calls (configurable, default: 12)
   - Before any parallel sub-agent spawn (detect from orchestrator pattern)
   - After receiving results from ≥2 parallel agents
3. **Implement** hook script: `.claude/hooks/auto-compress.sh` (or appropriate mechanism)
4. **Update** rule-010 to reference the automation (manual trigger remains as fallback)
5. **Document** in `docs/AI_REFERENCE.md` under "Context Management"

## Constraints

- Hook must NOT break session flow — compression should be advisory (log warning) not blocking
- Must work with both Claude Code CLI and API usage
- Token threshold should be configurable via env var `COMPRESS_THRESHOLD`

## Acceptance Criteria

- [ ] Hook script exists and is documented
- [ ] Triggers on configurable tool-call threshold
- [ ] rule-010 updated to reference automated trigger
- [ ] `docs/AI_REFERENCE.md` has "Context Management" section
- [ ] Fallback: manual `/compress-state` still works

## Copilot Compatibility

Not applicable — Copilot Chat is stateless per conversation. No compression mechanism exists or is needed.
