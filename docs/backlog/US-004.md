## Riepilogo

Auth flow: Volto → Plone login → `POST /api/v1/auth/plone-login` con Plone token → verifica via Plone `/users/@current` → JWT come httpOnly cookie (`ai_platform_token`). Payload JWT: `{sub, tenant_id, roles, exp, iss, type}`. Access token: 60min, refresh: 7 giorni. Cookie: httpOnly, SameSite=Strict.

**Residual risks (HIGH):**
- Auth events non ancora integrati con AuditService
- Refresh token rotation non implementata
- `tenant_id` viene da request body durante bootstrap (necessario, ma tenants devono pre-esistere)
